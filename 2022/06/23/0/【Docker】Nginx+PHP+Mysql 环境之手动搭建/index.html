【Docker】Nginx+PHP+Mysql 环境之手动搭建

分析

正常情况下, nginx环境和php环境是伴生的, 即有一个nginx环境就需要有一个php环境, (例外:php使swoole扩展后,可以自己启动http服务,不再需要nginx提供webService服务)

网上很多关于怎么搞一个镜像把nginx和php都装进去, 然后启动起来. 我个人认为这样并不是最佳的方案,一个容器只负责一个服务,这样可以最大限度进行解耦, 举个例子: 假如在使用中需要加装PHP扩展, 这时是不是只需要停止php容器装扩展,或者重新做一个镜像就ok,nginx的容里不需要再动了,反之亦然.

其实使用docker的composer可以实现对Docker容器集群的快速编排,不需要一个一个的再起docker容器了,不过这篇文章就是从原生的角度去搭建,以使我们更好的理解docker容器之间的关系和交互.

关键点

  1. nginx 和 php 需要各启容器,所以这里至少需要两个容器
  2. 需要使用容器数据卷将服务与数据分离
  3. nginx服务 和php服务(php-fpm) 是通过 fastcgi协议 进行,所以两个容器间需要进行网络交互,这里需要使用到docker的网络方面的知识
  4. 如果php需要对连接mysql数据库, 就需要安装mysql的相关扩展,我们这里使用的pdo,所以需要安装php-pdo扩展
  5. 本次的操作均为手动操作,不考虑自动扩容
